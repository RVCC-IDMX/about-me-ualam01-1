Blog Post: Using and Understanding CSS Animations

CSS animations is a useful tool that helps web developers in creating transitions between different states of various elements. It also enhances user engagement and provides visual feedback without the need of using Java Script! By using tools to define keyframes and specefying animation properties, developers can control the intermediate steps in animation sequences. Www.w3schools.com defines CSS animation as the following: "An animation lets an element gradually change from one style to another.You can change as many CSS properties you want, as many times as you want. To use CSS animation, you must first specify some keyframes for the animation. Keyframes hold what styles the element will have at certain times". There are many online blogs that help in understanding how CSS animation works!

MDN Web Docs gives an educational example of why one would prefer to use CSS animations over traditional methods: "They're easy to use for simple animations; you can create them without even having to know JavaScript. The animations run well, even under moderate system load. Simple animations can often perform poorly in JavaScript. The rendering engine can use frame-skipping and other techniques to keep the performance as smooth as possible. Letting the browser control the animation sequence lets the browser optimize performance and efficiency by, for example, reducing the update frequency of animations running in tabs that aren't currently visible."

What are Keyframes? Keyframes are used to implement CSS animation, developers define keyframes using the @keyframes rule. This rule specefies the styles that the element will have at various points during the animation. After defining the keyframes, the animation is applied to an element using the animation property. This property shortcut allows developers to set various aspects of the animation, such as its name, duration, timing function, delay, iteration count, direction, fill mode, and play state.

When working with CSS animations, applications we have utilized this semester, such as Visual Studio Code (VS Code), can enhance the development experience. VS Code offers features such as syntax highlighting, "IntelliSense" for CSS properties, and extensions that provide real-time previews of animations, aiding in efficient and accurate animation development.

Using CSS animations in VS Code is further enhanced by its debugging and version control capabilities. Developers can inspect their animations in real-time using browser developer tools alongside the Live Server extension to fine-tune their designs. Git integration allows for tracking changes to animation files, ensuring that previous iterations can be revisited if necessary. By combining these features, VS Code serves as a powerful platform for experimenting with and perfecting CSS animations, helping developers create visually stunning and interactive web pages.

When creating the code for the animation myself, it was far easier than what I was attempting to do. Reading about how to integrate CSS animation to your photos and seeing all the examples online was overwhelming, but the @keyframes tag made it very easy to grasp. I started out with writing extra code, and it was not working properly. Then, I condensed it and simplified it and voila... I got a rotating picture!
